# 3.6.接口

‍

## 接口的特点

* 接口用关键字interface修饰

* 类实现接口用implements表示

* 接口不能实例化

	接口如何实例化呢？参照多态方式，通过实现类对象实例化，这叫接口多态。

	多态的形式：具体类多态，抽象类多态，接口多态；

	多态的前提：有继承或者实现关系；有方法重写；有父（类/接口）引用指向（子/实现）类对象

* 接口的实现类

	要么重写接口中所有抽象方法

	要么是抽象类

‍

举例：创建一个`Jumpping`​的接口

```java
package com.pokes009;

public interface Jumpping {
	//我们发现abstract，方法是抽象的，既然是抽象的那么继承之后必须重写.
    public abstract void jump();
}

```

再创建一个Cat类实现接口`Jumpping`​

```java
package com.pokes009;

//定义了一个Cat类继承了Jumping接口，需要注意的是接口的继承关键字是：implements
//也就是Cat类实现接口Jumping

public class Cat implements Jumpping {
    @Override
    public void jump() {
        System.out.println("猫可以跳高");
    }
}
```

再创建一个测试类JumppingDemo

```java
package com.pokes009;

public class JumppingDemo {
    public static void main(String[] args) {

        //下面报错，说Jumpping是抽象的不能被实例化。知识点：接口也是一个抽象的内容
        //Jumpping j=new Jumpping();

        //使用多态的方法将其实例化！老规矩：编译看左边，执行看右边
        Jumpping j = new Cat();
        j.jump();   //猫可以跳高
    }
}

```

## 接口中成员变量的特点

* 接口中的成员变量只能是常量,并且被静态修饰，可以通过接口名直接访问。
* 接口中是没有构造方法，被实现之后所调用的构造方法是opject的构造方法。
* 接口中只能有抽象方法，不能有其他方法

注意：关于接口中的方法，JDK8和JDK9中有一些新的特性，后面在了解。

举例说明：

创建Inter接口，有成员变量num1，num2

```java
package com.pokes010;

public interface Inter {
    public int num1 = 10;
    public final int num2 = 20;
}
```

实现接口InterImpl类

```java
package com.pokes010;

public class InterImpl implements Inter {
    public InterImpl(){
        super();

        //接口中是没有构造方法的;
        //这里super()所调用的构造方法是opject的构造方法。
    }
}

```

测试类

```java
package com.pokes010;

public class InterfaceDemo {
    public static void main(String[] args) {


        //1.接口中的常量默认被final修饰，不能被重新赋值

        Inter i = new InterImpl();
        //下行我想给i.num1 = 30重新赋值，结果直接报错.没有被final修饰，但是也不能重新赋值
        //i.num1 = 30; 报错
    
        System.out.println(i.num1); //10
        System.out.println(i.num2); //20

        //2.可以通过接口名直接访问成员变量,说明成员变量默认是被静态修饰的。
        System.out.println(Inter.num1); //10

        //总结：接口中的成员变量只能是常量,并且被静态修饰，可以通过接口名直接访问。

    }
}

```

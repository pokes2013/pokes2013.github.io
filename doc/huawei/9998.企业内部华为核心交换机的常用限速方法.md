# 企业内部华为核心交换机的常用限速方法

## 给某一个VLAN做每IP限速

比如只给 **VLAN 20** 每个 IP 限速 5M/10M，其他 VLAN 不限

那就不能用 user-queue，必须用 **MQC 方式**

```bash
system-view

# 1. ACL 匹配 VLAN 20 的流量
acl number 3000
 rule permit ip source 192.168.20.0 0.0.0.255  # 写VLAN20网段
quit

# 2. 流分类
traffic classifier vlan20
 if-match acl 3000
quit

# 3. 流行为：每IP限速
traffic behavior perip_5m10m
 car cir 5120  per-address inbound   # 关键：per-address 每IP
 car cir 10240 per-address outbound  # 关键：per-address 每IP
quit

# 4. 流策略
traffic policy vlan20_limit
 classifier vlan20 behavior perip_5m10m
quit

# 5. 应用在【下联用户的物理口】
interface GigabitEthernet 1/0/1
 traffic-policy vlan20_limit inbound
 traffic-policy vlan20_limit outbound
quit
```

验证

```bash
# 查看流策略应用
display traffic policy applied

# 查看CAR统计
display qos car interface GigabitEthernet 1/0/1 inbound
display qos car interface GigabitEthernet 1/0/1 outbound

# 查看用户队列（每用户限速）
display qos user-queue interface GigabitEthernet 1/0/1
```



## VLAN整体共享50M

举例：

- 对 **VLAN 20** 整体限速 **50Mbps**
- 不管下面 10 人还是 20 人，**总共只能用 50M**
- 配置应用在 **核心交换机的下联物理口**（比如 GE1/0/1）



```bash
system-view

# 1. 流分类：匹配 VLAN 20
traffic classifier vlan20
 if-match vlan-id 20
quit

# 2. 流行为：整个 VLAN 限速 50M（共享）
traffic behavior limit_50m
 car cir 51200 green pass red discard
quit

# 3. 流策略
traffic policy vlan_limit
 classifier vlan20 behavior limit_50m
quit

# 4. 应用到下联用户的物理接口
interface GigabitEthernet 1/0/1
 traffic-policy vlan_limit inbound
 traffic-policy vlan_limit outbound   
quit
```



> 如果你只想限制下载，只配 outbound 即可。



## 疑问解答

> **VLAN整体共享50M，是vlan限速，为何要应用到物理接口？不是vlanif接口吗？**
>
> 
>
> **理论部分的回答：**
>
> 这是华为交换机最容易踩坑、但必须死记的一个关键点：
> 因为：VLAN 限速不能在 Vlanif 接口下做，必须在物理接口下做
> 我用最通俗、最核心的逻辑给你讲透，你以后再也不会懵。
>
> Vlanif 是三层逻辑接口，只负责路由、转发
> QoS 限速是二层硬件功能，必须在物理口（芯片）上执行
> 华为 S 系列核心交换机（S5700/S6700/S7700/S9300/S9700/S12700）：
> 不支持在 Vlanif 下调用 traffic-policy 做限速
> 敲了也不生效，甚至命令都不出来。
>
> **为什么必须绑在 “下联物理口”？**
>
> 你的流量路径是这样：电脑 → 下级交换机 → 核心交换机【物理口 GE1/0/1】→ 芯片 → Vlanif → 路由
>
> 限速要在流量刚进交换机的第一道关口拦截，
> 也就是：物理口 inbound
> 如果你绑在 Vlanif：流量已经走完芯片、进入系统层面；限速不精准、会丢包、会卡顿；华为硬件架构根本不支持
>
> **那绑在物理口，会不会影响其他 VLAN？**
>
> 不会！因为你流分类里写了：if-match vlan-id 20，交换机芯片会：
>
> 只对这个口里面属于VLAN20的流量，限速其他VLAN流量**不受影响**，这就是 VLAN 流控的标准做法。

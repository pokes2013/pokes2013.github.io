## 小白如何查看Linux的进程常用查看方法

标签：难点复盘

## 方法1：ps-ef用于找进程、看PID、杀进程

显示系统所有进程 + 完整详细信息

> p = Process（进程）s = Status（状态）
>
> -ef = 显示系统所有进程 + 完整详细信息 = every+full
>
> - -e= every（所有进程）意思是：列出系统里正在跑的全部程序，不只是当前用户的。
> - -f= full（完整格式）意思是：显示完整详细信息，包括：谁启动的（用户），PID（进程 ID），父进程ID，启动时间，完整命令行

```bash
ps -ef | grep ssh
```

你只是想 “找进程、看 PID、杀进程”，ps -ef 完全够用、最简洁、最通用。

运行结果：

```bash
pokes@pokes-cg:~/桌面$ ps -ef | grep ssh
root         695       1  0 13:51 ?        00:00:00 sshd: /usr/sbin/sshd -D [listener] 0 of 10-100 startups
pokes       1642       1  0 14:00 ?        00:00:00 /usr/bin/ssh-agent -s
pokes      57562   57183  0 14:28 pts/0    00:00:00 grep --color=auto ssh
```

出现三行之后，我就直接蒙了，哪一行才是我要找的进程。

> 规律（背下来，永远不会错）
>
> - 排除包含 grep 的行
> - 排除包含 ssh-agent 的行
>
> 剩下那一行，才是你要的！

按照上面的规则，剩下的就是第一行：

```bash
root  695  1  0 13:51 ?   00:00:00 sshd: /usr/sbin/sshd -D [listener] 0 of 10-100 startups
```

所以，你真正要关注的只有第一行：PID = 695 另外两行完全不用管、不能杀、杀了也没用。

| 列位置 | 内容          | 专业名称 | 含义说明                                  |
| ------ | ------------- | -------- | ----------------------------------------- |
| 1      | **root**      | UID      | 运行进程的用户（系统超级用户）            |
| 2      | **695**       | PID      | **进程 ID**（你要找的 PID，杀进程用这个） |
| 3      | **1**         | PPID     | 父进程 ID（1 = 系统启动的服务）           |
| 4      | **0**         | C        | CPU 占用率（0%）                          |
| 5      | **13:51**     | STIME    | 进程启动时间                              |
| 6      | **?**         | TTY      | 终端类型（?= 后台服务，无界面）           |
| 7      | **00:00:00**  | TIME     | 累计占用 CPU 时间                         |
| 8      | **sshd: ...** | CMD      | 进程命令（**SSH 服务监听进程**）          |

## 方法2：ps-aux显示CPU、内存 占用率 

显示所有进程（包含所有用户）+ 更详细的资源占用信息

> p = Process（进程）s = Status（状态）
>
> -a= all users（所有用户的进程）不是只看你自己，root、系统进程全都显示
> -u= user（显示用户、CPU、内存占用）能看到谁在跑、占多少 CPU、占多少内存
> -x= 没有终端的进程（后台进程也显示）比如 SSH、服务、守护进程，不会漏掉

运行结果：

```bash
pokes@pokes-cg:~/桌面$ ps -aux | grep ssh
root         695  0.0  0.1  15436  9224 ?        Ss   13:51   0:00 sshd: /usr/sbin/sshd -D [listener] 0 of 10-100 startups
pokes       1642  0.0  0.0   7980  2064 ?        Ss   14:00   0:00 /usr/bin/ssh-agent -s
pokes      97747  0.0  0.0  12324  2568 pts/0    S+   14:48   0:00 grep --color=auto ssh
```

| 位置   | 内容        | 专业含义                            |
| ------ | ----------- | ----------------------------------- |
| **1**  | `root`      | 运行进程的**用户**（超级用户）      |
| **2**  | `695`       | **PID 进程号**（你要找的就是它！）  |
| **3**  | `0.0`       | **CPU 占用率**                      |
| **4**  | `0.1`       | **内存占用率**                      |
| **5**  | `15436`     | 虚拟内存大小                        |
| **6**  | `9224`      | 物理内存大小                        |
| **7**  | `?`         | 无终端（后台服务）                  |
| **8**  | `Ss`        | 进程状态（S = 休眠，s = 主进程）    |
| **9**  | `13:51`     | 启动时间                            |
| **10** | `0:00`      | 占用 CPU 总时间                     |
| **11** | `sshd: ...` | **进程名称 / 命令**（SSH 服务本身） |



# ubuntu2024安装Cinnamon桌面



你已经通过 `sudo apt install cinnamon cinnamon-desktop-environment xrdp` 安装了 Cinnamon 桌面环境，但此时 XRDP 仍会默认启动之前的 Xfce4（或其他桌面）。需要修改 XRDP 的启动配置，让它启动 Cinnamon 会话。

---



```bash
sudo apt install cinnamon cinnamon-desktop-environment xrdp
```



## 步骤：让 XRDP 启动 Cinnamon

### 1. 编辑 XRDP 启动脚本

```bash
sudo nano /etc/xrdp/startwm.sh
```

找到文件末尾，**注释或删除**原有的所有启动命令（比如之前的 `startxfce4`），然后添加：

```bash
cinnamon-session
```

修改后的末尾应该类似：

```bash
# 原有内容全部注释掉
# test -x /etc/X11/Xsession && exec /etc/X11/Xsession
# exec /bin/sh /etc/X11/Xsession
# startxfce4

# 启动 Cinnamon
cinnamon-session
```

保存（`Ctrl+O`，回车，`Ctrl+X`）。

### 2. 重启 XRDP 服务

```bash
sudo systemctl restart xrdp
```

### 3. 从 Windows 远程桌面连接

- 打开远程桌面连接（`mstsc`）
- 输入 Ubuntu 的 IP 地址
- 输入你的 Ubuntu 用户名和密码
- 登录后应该直接进入 **Cinnamon 桌面**

---

## 可能出现的问题及解决

| 问题 | 解决方法 |
|------|----------|
| **登录后黑屏 / 只看到壁纸 / 鼠标卡死** | Cinnamon 需要硬件加速，但 XRDP 默认的 Xorg 后端可能不支持。尝试：<br>1. 安装 `cinnamon-session`：`sudo apt install cinnamon-session`<br>2. 改用 **VNC** 方式连接（见下方）<br>3. 或者换用 **MATE** 桌面（更兼容） |
| **登录后闪退回登录界面** | 查看 XRDP 日志：`sudo journalctl -u xrdp --since "5 minutes ago"`。常见原因是用户主目录权限问题：`sudo chown -R 你的用户名:你的用户名 /home/你的用户名` |
| **会话选择界面出现** | 如果你不想修改全局 `startwm.sh`，也可以在登录时手动选择：在 XRDP 登录窗口的“Session”下拉框中选择 `cinnamon`（前提是安装了 `xrdp-sesman` 并正确配置）。不过最简单还是修改 `startwm.sh` |

---

## 备选方案：使用 VNC 连接 Cinnamon（更稳定）

如果 XRDP 下 Cinnamon 表现不佳，推荐使用 **TigerVNC**：

```bash
# 安装 VNC 服务器
sudo apt install tigervnc-standalone-server

# 设置 VNC 密码
vncpasswd

# 启动 VNC 服务（使用 Cinnamon）
vncserver -localhost no :1 -geometry 1920x1080 -depth 24

# 在 Windows 上使用 VNC Viewer 连接 IP:1
```

如果 VNC 启动的不是 Cinnamon，编辑 `~/.vnc/xstartup`：

```bash
#!/bin/bash
cinnamon-session &
```

然后 `chmod +x ~/.vnc/xstartup`，重启 VNC 服务。

---

## 总结

- **修改 `/etc/xrdp/startwm.sh`** 加入 `cinnamon-session` → 重启 xrdp → 用 Windows 远程桌面连接。
- 若黑屏或不流畅，改用 **VNC**。
- Cinnamon 默认主题现代，无需额外美化，远程体验尚可。如果仍觉得卡，建议换回 Xfce4 并只做美化（前面提到的 Arc 主题 + Papirus 图标已经足够好看）。

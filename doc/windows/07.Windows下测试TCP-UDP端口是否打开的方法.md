# Windows下测试TCP/UDP端口是否打开的方法
在日常工作中，我们常常需要检测某台服务器的端口是否开放。除了使用网上的在线检测工具，在本地Windows电脑上也能通过简单工具完成检测，以下是两种常用方法的详细步骤：

## 一、使用tcping工具检测TCP端口是否开放
### （一）工具准备
1. 下载tcping工具；

   > 下载地址：https://download.elifulkerson.com/files/tcping/0.39/x64/tcping64.exe

2. 将下载得到的`tcping.exe`文件，复制到`C:\Windows\System32\`目录下。

### （二）检测操作
1. 打开Windows系统的命令行工具（cmd.exe）；
2. 输入命令 `tcping 地址 端口`（将“地址”替换为目标服务器的IP或域名，“端口”替换为需检测的TCP端口号），然后执行命令。

### （三）结果示例
检测freenat.club服务器的7000端口时，命令行输出如下，表明该端口开放：
```
C: Wsers\SWORD>tcping freenat.club 7000
Probing 167.88.162.106:7000/tcp - Port is open - time=193.882ms
Probing 167.88.162.106:7000/tcp - Port is open - time=200.130ms
Probing 167.88.162.106:7000/tcp - Port is open - time=199.662ms
Probing 167.88.162.106:7000/tcp - Port is open - time=221.075ms
Ping statistics for 167.88.162.106:7000
Approximate trip times in milli-seconds:
4 successful, 0 failed. (0.00% fail)
4 probes sent.
Minimum = 193.882ms, Maximum = 221.075ms, Average = 203.687ms
```

## 二、使用netcat工具检测UDP端口是否开放
### （一）工具准备
1. 下载netcat工具（推荐版本netcat-win32-1.12）；
2. 解压下载的工具包，将其中的`nc.exe`和`nc64.exe`两个文件，复制到`C:\Windows\System32\`目录下。

### （二）检测操作
1. 打开Windows系统的命令行工具（cmd.exe）；
2. 输入命令 `nc -vuz 地址 端口`（将“地址”替换为目标服务器的IP或域名，“端口”替换为需检测的UDP端口号），然后执行命令。

### （三）结果示例
检测freenat.bid服务器的7001 UDP端口时，命令行输出如下，表明该端口开放：
```
C: sers\SWORD>nc -vuz freenat.bid 7001
DNS fwd/rev mismatch: freenat.bid != 104.223.61.83.static.quadranet.com
freenat.bid [104.223.61.83] 7001 (?) open
```

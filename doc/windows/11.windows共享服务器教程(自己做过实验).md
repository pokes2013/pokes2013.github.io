# 公司文件服务器部署

## 1、需求分析

> 公司需求分析：
>
> - 共享名：share
>
> - 目录结构：D:\share\生产，D:\share\销售
>
> - 用户：user1、user2
>
> - 权限：user1：生产→读写，销售→只读；user2：销售→读写，生产→只读

## 2、共享设置（SMB共享）

- 新建共享目录share

- 右键 `D:\share` → **属性** → **共享** → **高级共享**

- 添加 `Everyone` 或 `Authenticated Users`
- 权限一律设为：**完全控制**

说明：共享权限只做 “放行”，精细权限靠 NTFS 控制

> Everyone 或 Authenticated Users这两个的关系是：
>
> - Everyone = 所有人，包括匿名访客、未登录用户
>
> - Authenticated Users = 只有已经成功登录的合法用户
>
> Authenticated Users 是 Everyone 的子集
>
> - 所有登录成功的用户 → 属于 Authenticated Users
>
> - 所有登录成功的用户 + 匿名 / 未登录用户 → 属于Everyone

## 3、NTFS权限配置（核心）

D:\share\生产

```
右键：D:\share\生产 → 安全 → 高级 → “禁用继承”
添加：Everyone 给读取权限、user1给写入权限
```

D:\share\销售

```
右键：D:\share\销售 → 安全 → 高级 → “禁用继承”
添加：Everyone 给读取权限、user2给写入权限
```

## 4、不弹登录框的解决办法

- 清理缓存:`net use * /delete /y`

- 更换凭证中相同IP的用户名和密码

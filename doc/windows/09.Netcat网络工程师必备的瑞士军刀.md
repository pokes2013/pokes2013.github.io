# Netcat（NC）：网络工程师必备的“瑞士军刀”
## 一、工具简介
Netcat 简称“NC”，是由霍斯特·克雷斯（Hobbit）于1995年开发的开源网络工具，体积小巧（仅49KB，经修改后可缩小至约10K且功能不减），支持 TCP/UDP 协议双向数据传输，被誉为渗透测试中的“瑞士军刀”。

它集多种功能于一体，可作为可靠的后端工具被其他程序或脚本驱动，同时也是功能丰富的网络调试和开发工具，能建立几乎任何类型的网络连接，广泛应用于网络运维、安全测试、渗透测试等场景，几乎所有 Linux 发行版都支持，是网络管理员、渗透测试人员和开发者的必备工具。

核心功能包括：
- 端口监听与扫描
- 远程文件传输
- 远程 shell 交互
- 网络连通性测试
- 代理转发、反弹 Shell 等

## 二、核心参数说明
### 1. 命令格式
- 主动连接目标：`nc [-options] hostname port[s] [ports] ...`
- 被动监听连接：`nc -l -p port [options] [hostname] [port]`

### 2. 常用参数
| 参数 | 功能说明 |
|------|----------|
| -d | 脱离控制台，后台运行 |
| -g | 源路由跳转点，最多8个 |
| -G | 源路由指针（4、8、12...） |
| -h | 查看帮助信息 |
| -i secs | 发送数据、扫描端口时的延迟间隔 |
| -l | 监听模式，用于接收入站连接 |
| -L | 强化监听，连接关闭后重新监听 |
| -n | 仅使用 IP 地址，不通过域名服务器（DNS）解析 |
| -o file | 将网络流量以十六进制形式保存到文件 |
| -p port | 指定本地端口号 |
| -r | 随机化本地和远程端口 |
| -s addr | 指定本地源地址 |
| -u | 启用 UDP 模式（默认 TCP 模式） |
| -v | 显示详细信息（使用两次 `-vv` 可显示更详细日志） |
| -w secs | 连接和最终网络读取的超时时间 |
| -z | 零 I/O 模式，仅用于端口扫描（不发送实际数据） |

### 3. 端口格式
- 单个端口：直接指定端口号（如 80）
- 端口范围：使用“m-n”表示（含首尾端口，如 80-8080）

## 三、常见用法（附实操命令）
### 1. TCP 端口测试
用于检测目标主机的 TCP 端口是否开放。
- 测试单一端口：`nc -vz 192.168.2.200 80`
- 测试连续端口（范围）：`nc -vz 192.168.2.200 80-8080`

### 2. UDP 端口测试
需通过 `-u` 参数启用 UDP 模式，检测目标主机的 UDP 端口连通性。
- 命令示例：`nc.exe -u -v -v 182.92.12.11 123`（双 `-v` 显示更详细的交互日志）

### 3. 网络连通性测试
通过服务器端监听端口、客户端连接的方式，验证两端网络连通性。
- 服务器端开启 TCP 监听（指定端口 80）：`nc -l -p 80`
- 服务器端开启 UDP 监听（指定端口 123）：`nc -u -l -p 123`
- 客户端连接服务器（需配合服务器端监听配置，输入对应 IP 和端口）

## 四、扩展应用示例
Netcat 可通过脚本实现自动化操作，例如批量定时关闭无线信号，避免非工作时间被非法枚举攻击（Windows 批处理脚本示例）：
```batch
@echo off
echo +++ Batch file to automatist telnet a telnet session. +++
SET statistics=%date:~0,4%-%date:~5,2%-%date:~8,2%
echo %statistics% >> c:\scripts\log\DisableWIFI-%statistics%.log
c:\scripts\nc.exe 10.10.177.247 23 < c:\scripts\DisableWIFI.txt >> c:\scripts\log\DisableWIFI-%statistics%.log
c:\scripts\nc.exe 10.10.179.130 23 < c:\scripts\DisableWIFI.txt >> c:\scripts\log\DisableWIFI-%statistics%.log
pause
```

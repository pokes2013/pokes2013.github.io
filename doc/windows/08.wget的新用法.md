## wget的更新用法



今天遇到一个问题wget下载问题，但是没下载下来.

遇到的问题是：没有下载下来，下载的过程貌似读取到内存中了！

```cmd

                    tion/octet-stream]...}
RawContentLength  : 296448

```

> 问AI说是：
>
> 你用 PowerShell 的`wget`（实际是`Invoke-WebRequest`的别名）执行命令后返回`StatusCode:200`，说明**文件已经成功下载到内存中**了，只是 PowerShell 默认不会自动把内存里的二进制内容保存为本地文件，所以你在 D 盘看不到`tcping64.exe`，并不是真的没下载成功。
>
> 同样我在CMD里面执行，还是这个样子。但是在git里面可以下载下来。

如果需要下载，正确的命令是：

```bash
wget https://download.elifulkerson.com/files/tcping/0.39/x64/tcping64.exe -OutFile tcping64.exe
```

> 注意：-OutFile 不区分大小写

## 代替方法使用curl

当然可以！现代 Windows 系统（Win10 1803 及以上、Win11）已经默认自带 curl 工具，无需额外安装，直接在 CMD 或 PowerShell 里就能用它下载文件，和 Linux/Mac 上的用法基本一致，用来下载 tcping64.exe 完全没问题；如果是更老的 Windows 版本，简单配置后也能使用。

```bash
curl -o tcping64.exe https://download.elifulkerson.com/files/tcping/0.39/x64/tcping64.exe
```

若想自定义保存路径，比如保存到桌面，可写：`curl -o C:\Users\你的用户名\Desktop\tcping64.exe 下载地址`。

> 注意：o是小写

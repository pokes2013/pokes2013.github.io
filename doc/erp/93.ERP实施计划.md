# ERP实施计划-无刷实施步骤

## 一、基础资料准备阶段

- [x] 前期调研
  - [x] 物料的种类数量：根据物料种类的数量，确定编码的位数；
  - [x] 物料属性：根据物料属性确定，勾选ERP中归属类别；
  - [x] 物料的编码规则：物料编码、名称、规格格式；
  - [x] 物料单位：BOM、仓库、生产、采购统一单位，或使用双单位；
  - [x] BOM结构：阶层划分与设计；
  
- [x] 编码规则
  
  - [ ] 编码规则编写，编写原则：技术部的俗称、别名、系别一定要和仓库实物匹配。
  - [ ] 开会发布，必须获得（技术、销售、生产、仓库、采购）多方认可，才可执行下一步基础资料的导入。
  
- [ ] 基础资料的录入（每周开会，数据说话）

  - [x] 成品的录入：每周最少20条记录（成品优先录入）
  - [x] 零部件的录入：每周最少50条
  - [x] BOM的录入：每周最少5个BOM
  - [x] 包材的信息收集：因为前期工程没有整理包材资料，必须每个BOM即时提供包材数据；
  - [x] 对于编码规则的认可，必须技术、销售、生产、仓库、采购全部认可。
  - [ ] 工序档案
  - [ ] 工艺路线

- [ ] 操作手册的编写

  

- [x] 销售订单录入

  - [ ] 根据技术部成品录入的情况，录入销售订单。
  - [ ] 仅有成品物料编码、没有BOM的，销售录入订单之后，由生产计划安排纸质加工单生产，成品入库在系统里面入，即：销售下单和成品仓发货、销售发货单，在系统内操作。中间的采购、生产都在线下先跑。




- [x] 流程的确定
  - [x] 和销售对接；
  - [x] 和控制板对接；
  - [x] 项目号编码规则；
  - [x] 料号的编码规则；
  
  
  
- [x] 项目评审单、开会评审完之后，交于技术部做BOM，需要带上项目号、料号；

- [x] 工程部提供包材数据；

- [x] 技术部BOM制作，没有项目号和料号拒收；

- [x] 销售订单录入：销售根据成品物料的项目号、录入《销售订单》；

- [x] 生产计划：计划排产，根据销售订单执行《MRP运算》投放《生产加工单》和《采购申请单》

- [x] 采购处理《采购申请单》，即：采购查询库存、根据库存情况，行关闭部分物料；生产《采购订单》；

- [x] 采购根据《采购订单》跟供应商交流，确认交期、单价，给供应商下单；

- [x] 供应商送货，经入库检验合格后，由原材料仓库根据采购订单号做《采购入库》生成原材料库存；

- [ ] ==《采购入库》审核后，自动生成《应付单》==

- [x] 生产审核《生产加工单》，并打印最少2份。1份交于物料员，1份交与线长确认生成数量；

- [x] 产线物料员根据《生产加工单》到仓库领取物料。双方签字确认；

- [ ] 通用的工序《工艺路线》

- [ ] 工程进行《工序排程》；

- [x] 生产主管根据《生产加工单》进行生产；

- [x] 产线做出成品，经品质检验合格后，需要做《成产品入库》操作，由成品仓管审核后，生成成品库存。

- [x] 销售根据订单交期填写《发货单》，发货单审核后会自动减掉成品库存；

- [x] ==《发货单》审核后自动生成财务《应收单》==

- [ ] 客户付款后，财务需要做《收款单》，点击取数关联《发货单》

- [ ] 月底财务需要做原材料的《付款单》，点击取数关联《采购入库单》

- [ ] 至此，此单归档，用于后续查询。

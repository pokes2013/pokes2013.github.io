# JS的三种引入方式完整详解

你想学习 JavaScript 的核心引入方式，这是 JS 入门的核心基础，JavaScript 代码在 HTML 页面中有 **三种标准、常用的引入方式**，分别是：**行内引入、内部引入、外部引入**，三种方式各有适用场景，下面逐一详细讲解，所有示例可直接复制运行。

## 前置说明

JavaScript 是**脚本语言**，它的代码必须依赖 HTML 页面才能运行，浏览器在解析 HTML 时，遇到 JS 代码会调用 JS 引擎执行，这也是所有引入方式的前提。

------

## 一、方式一：行内引入

### ✅ 语法特点

将 JavaScript 代码**直接写在 HTML 标签的事件属性**中，是**最原始、最基础**的引入方式。

核心依赖：HTML 标签的**事件属性**（以`on`开头的属性），最常用的是 `onclick`（点击事件）、`onmouseover`（鼠标悬浮事件）等。

### ✅ 代码示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>行内引入JS</title>
</head>
<body>
    <!-- 点击按钮，执行行内的JS代码：弹窗提示 -->
    <button onclick="alert('你点击了按钮，这是行内JS效果')">点我试试</button>
    <!-- 鼠标悬浮到div上，执行行内的JS代码：弹窗提示 -->
    <div onmouseover="alert('鼠标悬浮上来了')" style="width:100px;height:100px;background:red;">悬浮我</div>
</body>
</html>
```

### ✅ 优缺点总结

✅ **优点**：书写简单、耦合度高，想给哪个标签加交互效果，直接写在标签上即可，适合**简单的、单一的交互逻辑**。

❌ **缺点**：

1. 违反 **「结构与行为分离」** 的前端开发规范（HTML 负责结构，JS 负责行为，混写在一起不规范）；
2. 代码复用性极差，多个标签要实现相同效果，需要重复写代码；
3. 逻辑复杂时，行内代码会变得臃肿、难以维护。

### ✅ 适用场景

只需要实现**简单的、一次性**的交互效果，比如单个按钮的点击弹窗、单个元素的简单事件响应。

------

## 二、方式二：内部引入

### ✅ 语法特点

将所有 JavaScript 代码，**统一写在 HTML 页面的 `<script>` 标签内部**，`<script>` 标签是专门承载 JS 代码的标签，无内容时也可自闭合（但一般不推荐）。

### ✅ 核心语法

```html
<script>
    // 这里写所有的JavaScript代码
    JS代码;
</script>
```

### ✅ 代码示例

html



预览









```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>内部引入JS</title>
    <!-- 可以写在head中 -->
    <script>
        console.log("我是写在head里的内部JS代码");
    </script>
</head>
<body>
    <button id="btn">点击弹窗</button>

    <!-- 推荐：写在body标签的末尾 -->
    <script>
        // 完整的JS逻辑，定义变量、函数、绑定事件都可以
        var str = "我是内部引入的JS";
        console.log(str);
        // 给按钮绑定点击事件
        document.getElementById("btn").onclick = function(){
            alert(str);
        }
    </script>
</body>
</html>
```

### ✅ 重要注意点：`<script>` 标签的位置

<script> 可以写在 head 里，也可以写在 body 里，强烈推荐写在 <body> 标签的最后一行（</body> 前面），原因：

> HTML 页面是**从上到下依次解析**的，如果 JS 写在 head 里，会先加载 JS 代码，再加载页面结构；如果 JS 需要操作页面元素（比如获取按钮），此时元素还没加载出来，会执行失败！

### ✅ 优缺点总结

✅ **优点**：

1. 实现了「结构与行为分离」，HTML 写结构，JS 写在统一的标签里，代码更整洁；
2. 同一个页面的所有 JS 逻辑都集中管理，复用性比行内引入高很多；
3. 无需额外请求文件，加载速度快。
4. JS 代码和 HTML 页面**耦合在一起**，只能在当前页面使用，无法被其他页面复用；
5. 页面逻辑复杂时，JS 代码会很多，导致 HTML 文件体积过大，不利于维护。

### ✅ 适用场景

**中小型项目、单个页面**的交互逻辑开发，比如表单验证、页面简单动画、局部元素交互等，是日常开发中最常用的方式之一。

------

## 三、方式三：外部引入（推荐）

### ✅ 语法特点

这是 **JavaScript 最核心、最推荐的引入方式**！核心逻辑：

1. 创建一个**独立的 `.js` 后缀的纯 JavaScript 文件**，所有 JS 代码都写在这个文件中

   > 注意⚠️：**`.js文件中不能写<script>标签`**，只写纯代码；

2. 在 HTML 页面中，通过 `<script src="JS文件路径"></script>` 标签，引入这个外部 JS 文件。

### ✅ 核心语法

#### 步骤 1：创建外部 JS 文件（例如：`index.js`）



```javascript
// index.js - 纯JS代码，无任何HTML标签
console.log("我是外部引入的JS文件代码");
// 定义函数
function showMsg(){
    alert("外部JS文件执行成功！");
}
// 页面加载完成后绑定事件
window.onload = function(){
    document.getElementById("btn").onclick = showMsg;
}
```

#### 步骤 2：在 HTML 中引入该 JS 文件

html



预览









```
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>外部引入JS</title>
</head>
<body>
    <button id="btn">点击调用外部JS的方法</button>

    <!-- 引入外部JS文件，核心：src属性写文件路径 -->
    <script src="./index.js"></script>
</body>
</html>
```

### ✅ 关键注意事项（必看）

1. `<script>` 标签的 `src` 属性：专门用于指定外部 JS 文件的**路径**（相对路径 / 绝对路径都可以）；

2. **引入外部 JS 的`<script>`标签，必须是单标签，不能写闭合标签包裹内容**：

   

   ❌ 错误写法：

   html

   

   预览

   

   

   

   

   ```
   <script src="./index.js">
       // 这里的代码不会执行！！！
       alert("无效代码");
   </script>
   ```

   

   ✅ 正确写法：如果需要额外写内部代码，新增一个独立的

   ```
   <script>
   ```

   标签即可

   html

   

   预览

   

   

   

   

   ```
   <script src="./index.js"></script>
   <script>
       alert("这是内部代码，会正常执行");
   </script>
   ```

   

3. `.js` 文件中**只能写纯 JavaScript 代码**，不能包含`<script>`、`<html>`等任何 HTML 标签，否则会报错。

### ✅ 优缺点总结

✅ **优点（核心，为什么推荐！）**：

1. **完全实现「结构、样式、行为三者分离」**：HTML (结构) + CSS (样式) + JS (行为) 三个文件各司其职，符合前端工程化开发规范；
2. **极致复用**：一个外部 JS 文件，可以被**多个 HTML 页面引入复用**，比如多个页面都需要表单验证，只需要写一次 JS，所有页面引入即可，极大减少重复代码；
3. **便于维护**：JS 代码集中在独立文件中，修改逻辑时只需改 JS 文件，无需动 HTML 页面，大型项目中能大幅提升开发效率；
4. **浏览器缓存**：外部 JS 文件被加载一次后，会被浏览器缓存，下次打开页面时无需重新加载，提升页面加载速度。
5. 需要**额外发起一次 HTTP 请求**加载 JS 文件（如果引入多个 JS 文件，会增加请求数）；
6. 路径写错时，会导致 JS 文件加载失败，代码无法执行（排查即可解决）。

### ✅ 适用场景

**所有场景，尤其是中大型项目、多页面开发**！这是企业开发中的**标准引入方式**，是你必须掌握并优先使用的方式。

------

## 四、三种引入方式的优先级（执行顺序）

如果一个 HTML 页面中，同时使用了三种引入方式，**代码的执行顺序遵循：HTML 页面的解析顺序 → 从上到下，依次执行**

补充优先级规则：

1. 行内 JS：绑定在标签上，**触发对应事件时才执行**（比如点击按钮）；
2. 内部 JS 和外部 JS：写在页面哪个位置，就哪个时候执行，谁在上方谁先执行；
3. 优先级补充：**行内 JS 的执行优先级最高**（因为是事件触发，主动执行），但加载优先级由书写位置决定。

------

## ✨ 总结（核心必记）

1. **行内引入**：写在 HTML 标签的事件属性里，简单但不规范，适合极简交互，**尽量少用**；

2. **内部引入**：写在 HTML 的`<script>`标签内，结构分离，适合单页面开发，**常用**；

3. **外部引入**：写在独立`.js`文件，通过`<script src="">`引入，**完全分离、极致复用、便于维护**，**推荐优先使用**，企业开发标准方式。

4. 核心开发原则：**能外部引入，就不用内部引入；能内部引入，就不用行内引入**，遵循「结构与行为分离」，让你的代码更专业、更易维护！

5. 关键口诀：`<script>`标签是 JS 的容器，内部写代码，外部引文件，位置在 body 末尾最优！
